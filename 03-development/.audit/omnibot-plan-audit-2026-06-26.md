## 2. Correctness Audit

### F1 (Medium) — `phase7_plan.md` 未引用 `phase8_doc_gen.py` 鉤子

**Symptom**:P7 plan 描述「advance-phase to P8」時,**沒有提到** P7→P8 advance 會自動觸發 `phase8_doc_gen.generate()`(commit `51bd4a8` 引入)。

**對齊 commit**:`51bd4a8` review-fix #3:
> Hook scripts.phase8_doc_gen.generate() into cmd_advance_phase when next_phase == 8

**影響**:
- 執行者看到 P7 plan,以為 advance-phase 是唯一動作
- 實際上 framework 會自動產生 `08-config/CONFIG_RECORDS.md` + `RELEASE_CHECKLIST.md`,然後 commit
- 若執行者手動在 P8 跑 `python3 scripts/phase8_doc_gen.py --project .`,會 overwrite 已被 commit 的版本(無害,但易混淆)

**建議修法**(plan-only,non-workaround):
在 P7 plan 的 advance-phase 步驟之後加:
```markdown
> **Auto-trigger on P7→P8 advance**:
> `advance-phase --completed 7` 會自動呼叫 `scripts/phase8_doc_gen.py --project .`
> 產生 `08-config/CONFIG_RECORDS.md` + `RELEASE_CHECKLIST.md`(commit `4738542`)。
> 若 generator 失敗,advance 仍 SUCCESS;warning 提示手動重跑。
```

**驗證**:讀 `phase7_plan.md` grep `phase8_doc_gen` → 0 hit;讀 `harness/harness_cli.py` advance-phase → 確認 hook 已實作。

**Status**:**NOT WORKAROUND** — Plan 應反映 framework 行為;執行者不該從原始碼反向發現 framework 在 advance-phase 偷偷做了什麼。

### F2 (Low) — `phase8_plan.md` 提及 `CONFIG_RECORDS.md` 但無 deterministic baseline 步驟

**Symptom**:P8 plan tag `[CONFIG-RECORDS]` / `[RELEASE-CHECKLIST]` 描述「Generate」,但沒區分「framework 自動產生」vs「LLM agent 補完」。

**對齊 commit**:`4738542` phase8 deterministic:
> Replace with a deterministic Python script
> Phase 8 reviewer (LLM) keeps the task of reading the script output, flagging any missing sections, and appending human-only context.

**影響**:
- 執行者讀 P8 plan,可能誤以為要從零寫 `CONFIG_RECORDS.md`(舊 LLM-agent 行為)
- 實際上 framework 已經 deterministically 產生 baseline;LLM 只需要審閱 + append human-only context

**建議修法**(plan-only):
```markdown
> **[CONFIG-RECORDS]** baseline 已經由 `phase8_doc_gen.py` deterministically 產生
> (見 P7 plan §Auto-trigger on P7→P8 advance)。本 phase 工作是:
> 1. 讀 baseline → flag missing sections
> 2. Append human-only context(不可由 generator derive 的部分)
> 3. 不要 overwrite framework 產生的版本(會破壞 determinism)
```

**Status**:**DOCUMENTATION DRIFT** — 不影響功能,但會誤導執行者重做 generator 已做的工作。
